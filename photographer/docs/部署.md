# 摄影师预约小程序 — 后端部署文档

> **目标**:把 `photographer/backend` 部署到京东云服务器 `111.228.10.86`,通过 `xinweijia.net` 域名对外提供 `/api/pm/*` 接口和 `/admin/` 运营后台,与现有的成绩分析后端共存。
>
> **架构**:
>
> ```
> 用户 ──HTTPS──> nginx (443) ──┬── /api/pm/      → 8001 (摄影师后端)
>                              ├── /admin/        → 8001 (运营后台 SSR)
>                              ├── /pm-uploads/   → 8001 (本地图片占位,接通 OSS 后改走 CDN)
>                              ├── /api/         → 8000 (成绩分析后端,已有)
>                              ├── /uploads/     → 8000 (成绩分析上传,已有)
>                              ├── /app/         → 静态站 (成绩分析前端,已有)
>                              └── /            → 7861 (aipic,已有)
> ```
>
> **端口分配**:
>
> | 服务 | 端口 | 进程 |
> |---|---|---|
> | 摄影师预约后端 | **8001** | systemd `photographer-backend.service` |
> | 成绩分析后端 | 8000 | systemd `grade-analysis.service`(已有) |
> | aipic | 7861 | (已有) |
> | nginx | 80/443 | (已有) |

### 2.3 一键部署

```bash
cd /data/NviHo/photographer/backend

`deploy_server.sh` 会自动完成:

1. **建/更新 venv** — `python3 -m venv venv` + `pip install -r requirements.txt`
2. **准备 `.env`** — 首次自动 `cp .env.example .env`(默认占位模式可直接跑)
3. **数据库初始化** — 用 SQL 查 `admin` 账号是否存在,不存在就跑 `seed_data.py`
4. **安装 systemd 服务** — `cp photographer-backend.service /etc/systemd/system/` + enable
5. **启动并自检** — `systemctl restart` + `curl http://127.0.0.1:8001/api/pm/health`

成功标志:

```

## 三、配置 Nginx (首次部署的最后一步)

### 3.1 实际位置

服务器上的 nginx 站点配置文件:

```
/etc/nginx/sites-available/aipic          ← 实际生效的配置
/etc/nginx/sites-enabled/aipic            ← 软链接 -> sites-available/aipic
```

完整版本保存在项目里:[`backend/nginx.aipic.conf`](../backend/nginx.aipic.conf)

### 3.2 修改步骤

```bash
# 备份 (改坏了能恢复)
cp /etc/nginx/sites-available/aipic /etc/nginx/sites-available/aipic.bak.$(date +%Y%m%d)

# 编辑
vim /etc/nginx/sites-available/aipic
```

在 **443 server 块** 的 `client_max_body_size 100M;` 下方, **`location /api/` 上方** 插入这 3 段:

```nginx
    location /api/pm/ {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        client_max_body_size 20M;
    }

    location /admin/ {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        client_max_body_size 20M;
    }

    location /pm-uploads/ {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        expires 7d;
    }
```

> ⚠️ 关键: `/api/pm/` 必须在 `/api/` 上面,否则会被 `/api/` 截胡到 8000 端口的成绩分析后端。

### 3.3 验证 + 重载

```bash
nginx -t                   # 看到 "syntax is ok" + "test is successful" 才能继续
systemctl reload nginx     # 平滑重载,不断现有连接
```

如果 `nginx -t` 报错,直接看错误行号回 vim 改正,改对了再 reload。

## 八、出问题怎么排查

### 问题 1: `./deploy_server.sh` 卡在 pip install

国内访问 PyPI 慢。临时换成清华镜像:

```bash
venv/bin/pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题 2: `systemctl restart photographer-backend` 启动失败

```bash
journalctl -u photographer-backend -n 50 --no-pager
```

常见原因:
- **`bcrypt` 版本错** — 重跑 `./deploy_server.sh` 会按 `requirements.txt` 装对版本(< 4.1)
- **`.env` 缺失** — `cp .env.example .env`
- **8001 端口被占用** — `ss -tnlp | grep 8001` 看是谁占了
- **数据库 schema 不一致**(改过 model 后) — 删 db 重跑 seed:`rm photographer.db && venv/bin/python seed_data.py`

### 问题 3: 接口能 curl 但浏览器 404

检查 nginx location 顺序,`/api/pm/` 必须在 `/api/` 上方:

```bash
nginx -T 2>/dev/null | grep -A2 "location /api"
```

如果 `/api/` (8000) 在 `/api/pm/` 上面,会优先匹配,所有 `/api/pm/*` 请求都被发到成绩分析后端,导致 404。

### 问题 4: 上传图片后小程序展示不出来

检查图片 URL:

- ✅ 应该是 `https://www.xinweijia.net/pm-uploads/work/20260430/abc.jpg`
- ❌ 不应该是 `https://www.xinweijia.net/uploads/...`(那是成绩分析的)

如果是 `/uploads/` 开头,说明后端路径还没改干净,搜索代码里所有 `/uploads/` 改为 `/pm-uploads/`。

### 问题 5: 服务器磁盘不够

按方案 §7.5 清理 CordC + xfce4 + VNC,释放约 5GB:

```bash
# 看磁盘大头
du -h --max-depth=2 /root /data 2>/dev/null | sort -hr | head -20

# 卸载桌面环境
apt remove --purge xfce4 xfce4-goodies tigervnc-standalone-server
apt autoremove

# 清 apt 缓存
apt clean

# 清 journal 日志(只保留最近 7 天)
journalctl --vacuum-time=7d
```
