# 部署指南

> 目标: 把摄影师预约后端从 Mac 推到 GitHub, 再到京东云服务器 (`111.228.10.86`, `xinweijia.net`) 跑起来。
>
> 与现有的成绩分析后端共存: 成绩分析跑 8000 端口, 本项目跑 **8001 端口**, Nginx 按 location 分流。

## 二、服务器: 拉代码 + 部署

### 2.1 SSH 上服务器

```bash
ssh root@111.228.10.86
```

### 2.2 拉新代码

服务器在国内, GitHub 需要代理:

```bash
cd /data/NviHo
git -c http.proxy=http://127.0.0.1:7890 -c https.proxy=http://127.0.0.1:7890 pull
```

如果 `/data/NviHo` 目录不存在(首次部署), 先 clone:

```bash
cd /data
git -c http.proxy=http://127.0.0.1:7890 -c https.proxy=http://127.0.0.1:7890 clone <你的 GitHub 仓库 URL> NviHo
```

### 2.3 一键部署后端

```bash
cd /data/NviHo/photographer/backend

# 国内镜像装 pip 包不用代理, 但 venv 里如果之前设过代理变量先取消
unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY

chmod +x deploy_server.sh
./deploy_server.sh
```

脚本会自动:

1. 创建/更新 Python venv
2. `pip install -r requirements.txt`
3. 首次复制 `.env.example` → `.env`(默认占位模式可直接跑)
4. 数据库为空时跑 `seed_data.py` 初始化品类和示例摄影师
5. 安装 systemd 服务 `photographer-backend.service`
6. 启动并自检 `curl http://127.0.0.1:8001/api/pm/health`

成功后看到:

```
✓ photographer-backend 运行中
{"status":"ok","service":"photographer-backend"}
```

### 2.4 配置 Nginx 反代

编辑现有的 nginx 站点配置 (一般在 `/etc/nginx/sites-available/xinweijia.net.conf` 或 `/etc/nginx/conf.d/xinweijia.conf`), 把 [`backend/nginx.example.conf`](../backend/nginx.example.conf) 里的 3 个 location 块加到 server 块内 **`/api/` 上方**(因为 `/api/pm/` 必须比 `/api/` 优先匹配):

```nginx
location /api/pm/ {
    proxy_pass http://127.0.0.1:8001;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    client_max_body_size 20M;
}

location /admin/ {
    proxy_pass http://127.0.0.1:8001;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    client_max_body_size 20M;
}

location /uploads/ {
    proxy_pass http://127.0.0.1:8001;
    proxy_set_header Host $host;
    expires 7d;
}

location /api/ {
    proxy_pass http://127.0.0.1:8000;   # 成绩分析后端, 已有, 不动
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}
```

测试 + reload:

```bash
nginx -t          # 语法检查
systemctl reload nginx
```

## 五、上线前必改的安全配置

> ⚠️ 当前默认配置只适合开发期。正式对外服务前必须改:

| 项 | 文件 | 默认值 → 改为 |
|---|---|---|
| 管理员密码 | 数据库 | `admin/admin` → 复杂密码(在 admin 后台改, 或 SQL 直改) |
| `SECRET_KEY` | `.env` | `please-change-me-in-production` → 32 位随机串 (`openssl rand -hex 32`) |
| 关闭 dev-login | `app/routers/auth.py` | 上线前注释掉 `dev_login` 路由 |
| CORS allow_origins | `app/main.py` | `["*"]` → 仅小程序域名 |

## 六、出问题怎么排查

### 服务启动失败

```bash
journalctl -u photographer-backend -n 50 --no-pager
```

常见原因:
- `bcrypt` 版本错: 重跑 `./deploy_server.sh` 会按 `requirements.txt` 装对版本
- `.env` 缺失: 复制 `.env.example`
- 8001 端口被占用: `ss -tnlp | grep 8001` 看是谁占了

### 接口能访问但返回 404

检查 nginx 配置: `/api/pm/` 必须在 `/api/` **上面**, 否则会被 `/api/` 截胡到 8000 端口。`nginx -T | grep -A2 "location /api"` 看顺序。

### 服务器磁盘不够

按方案 §7.5 清理 CordC + xfce4 + VNC, 释放约 5GB:

```bash
# 看大头是谁
du -h --max-depth=1 / | sort -hr | head -20
# 卸载桌面
apt remove --purge xfce4 xfce4-goodies tigervnc-standalone-server
apt autoremove
# 清 apt 缓存
apt clean
```
